## Brief

Generates a daily todo file by aggregating pending work across all active projects. Scans runtime-notes for recent session activity, checks existing todo files for unchecked items, and reads MEMORY.md indexes for project activity hints. Output: `~/.claude/todo-YYYY-MM-DD.md`.

## Step 0: Load Shared Guidelines and Runtime Context

Read `~/.claude/skills/GUIDELINES.md` before proceeding. Apply all rules — forbidden paths,
retry logic, tool preferences, verbosity, timeouts, post-run insights, and the file lock
protocol — for the entire duration of this skill run.

Also read `~/.claude/skills/runtime-notes.md` for past run history relevant to this skill.
If it does not exist yet, continue without it.

> Lock reminder: acquire a lock via `~/.claude/skills/shared/lock-file.sh acquire` before
> every Edit/Write, and release it immediately after. Never write to `runtime-notes.md` or
> any SKILL.md without holding its lock.

## Usage

```
/daily-todo [YYYY-MM-DD]
```

| Argument | Type | Description |
|---|---|---|
| `YYYY-MM-DD` | Optional | Target date. Defaults to today. |

## Phase 1 — Determine Date & Check Existing

1. Parse the argument for a date. Default to today (`YYYY-MM-DD`).
2. Set output path: `~/.claude/todo-YYYY-MM-DD.md`.
3. If the file already exists, read it and note which items are checked/unchecked. These carry forward.

## Phase 2 — Scan All Projects

### 2.1 Discover active projects

Scan for `.claude/` directories in these known project roots (do NOT glob from `~/`):

```
~/Code/Personal/home-server/
~/Code/Versable/enhancement-product/frontend/
~/Code/Versable/enhancement-product/backend/
~/Code/Versable/v2-enhancement-product/
~/Code/Versable/scripts/
~/Code/Claude/visualize-claude/
~/Code/Claude/game-theory/
~/Code/Claude/resumes/
~/Code/Personal/helper-docs/
```

For each project that has a `.claude/` directory, it's considered active.

### 2.2 Scan runtime-notes

For each active project, read `.claude/skills/runtime-notes.md` (first 30 lines only — recent entries are at top).

Extract:
- Sessions from yesterday and today (match `## session:` or `## skill-name:` headers with dates)
- Any mentions of "TODO", "pending", "follow-up", "next step", "blocked"
- The purpose line from each recent entry

### 2.3 Check existing todo files

Read `~/.claude/todo-*.md` files (most recent 3 only). Extract unchecked items (`- [ ]`).

### 2.4 Scan MEMORY.md indexes

For each active project, check if a project-scoped MEMORY.md exists at:
`~/.claude/projects/-{encoded-path}/memory/MEMORY.md`

Read each index for project-type memories that suggest active work (look for keywords like "active", "in progress", "blocked", "pending").

## Phase 3 — Generate Todo

Write the todo file to `~/.claude/todo-YYYY-MM-DD.md`:

```markdown
# Todo — <Month Day, Year>

## Carried Forward

<Unchecked items from previous todo files, grouped by project>

## Recent Activity

<For each project with recent runtime-notes entries:>
### <project-name>
- Last session: <date> — <purpose>
- <Any pending items extracted>

## Suggested Tasks

<Based on patterns: stale memories, projects with no CLAUDE.md, large runtime-notes needing archival, etc.>

---

*Generated by /daily-todo — <timestamp>*
```

### 3.1 Prioritization rules

- **P0 (Critical):** Items marked as blocked or time-sensitive in runtime notes
- **P1 (Active):** Unchecked items from yesterday's todo that are still relevant
- **P2 (Suggested):** Inferred from stale project state (no CLAUDE.md, large runtime-notes, etc.)

## Phase 4 — Report

Print a compact summary:

```
  ✓ Daily todo generated
  File:     ~/.claude/todo-YYYY-MM-DD.md
  Projects: N scanned, M with recent activity
  Items:    X carried forward, Y new
```

## Phase 5 — Post-Run

Write a runtime note via:

```bash
bash ~/.claude/skills/shared/prepend-runtime-note.sh "daily-todo" /tmp/runtime-note-entry.md
```

---

## Notes

- This skill reads across many projects but writes only to `~/.claude/`. It does not modify any project files.
- The project list in Phase 2.1 is hardcoded. If new projects are added, update this list. A future enhancement could auto-discover projects from `~/.claude/projects/`.
- Runtime notes are read with a 30-line limit per project to keep context manageable.
- The skill is designed to be run once per day, typically at the start of a session.
