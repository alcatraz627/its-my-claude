---
name: Skip Learn by Doing for simple tasks
description: User does not want to be asked to write code themselves for straightforward/utility tasks — just do it
type: feedback
---

Don't use "Learn by Doing" prompts for simple, mechanical tasks like writing shell scripts, config files, or utility code. Just write it directly.

**Why:** User finds it frustrating to be asked to implement something when the answer is obvious and they just want it done.

**How to apply:** Reserve Learn by Doing for genuinely interesting design decisions in core application logic, not boilerplate or infrastructure scripts. When in doubt, just do the work.
