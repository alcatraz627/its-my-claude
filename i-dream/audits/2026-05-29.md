Every project centralizes env var access: a config module, a validated schema (zod, pydantic), or a typed wrapper. Before adding a raw env read, grep how existing vars are read.

```bash
rg -n 'process\.env\.|os\.getenv|os\.environ' src/
```

If a central config module exists, route through it. Raw reads scattered through modules create drift and bypass validation.

Graduated from atone slug `adding-env-var-reads-without-checking-the-project-s-config-pattern` (S3, 2×).
